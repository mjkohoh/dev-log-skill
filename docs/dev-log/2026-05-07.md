### Decisions
- 默认日志位置确定为 `docs/dev-log/YYYY-MM-DD.md`，按天追加记录。
- 日志默认可以提交到 git，因此必须执行去重、价值判断和敏感数据脱敏。
- 默认读者风格为 `hybrid`，兼顾人类阅读体验和 agent 接力结构化上下文。
- 默认触发策略为 `conservative`，只在用户明确要求、关键决策、阻塞、任务完成或上下文切换等高价值节点记录。
- 支持项目级配置，优先读取 `.devlog.yml`，其次读取 `docs/dev-log/config.yml`；无配置时使用保守默认值。
- 写入策略采用 append-only，旧记录需要补充或纠错时追加 update/correction 条目，而不是静默修改历史。
- `evals/` 定位为验收样例和质量保障，不是运行 skill 的必需部分，也不是 Codex 专用运行入口。
