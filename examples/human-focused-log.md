# Dev Log - 2026-05-07

## 2026-05-07 14:10 CST - 登录限流方案调整

Agent: Codex
Status: done

### Context
本轮在实现登录失败次数限制。原计划先用进程内 Map 保存失败计数，但检查部署方式后发现服务会多实例运行，内存计数会导致不同实例之间状态不一致。

### Decisions
改用 Redis 记录限流状态。key 按 `login:fail:{user_id}` 组织，失败后设置 15 分钟过期时间。这样多个实例可以共享限流状态，也避免手动清理历史记录。

### Changes
实现集中在认证模块和测试用例中。登录失败时写入 Redis，登录成功后清理对应 key。

### Verification
已运行认证相关测试。登录失败累计、过期释放、登录成功清理都通过。

### Next Steps
后续如果要支持 IP 维度限流，可以复用同一套 Redis key 策略，但需要单独评估误伤风险。
